## 1. 実装済みの実態（コードを読んで確認）

| 要件v2 | 実装状況 | 場所 |
| --- | --- | --- |
| `HIGH_SAMPLING_RATE_SENSORS` 権限 | ✅ 宣言済み | `AndroidManifest.xml` |
| IMU高レート収録（FASTEST＋失敗時GAMEフォールバック） | ✅ | `ImuRecorder.java` |
| 実効サンプリングHz測定 | ✅ `getSampleRateHintHz()` | `ImuRecorder.java` |
| VFE特徴（バーストON窓のRMS中央値） | ✅ `computeAccelRmsOnMedian()` | `ImuRecorder.java` |
| バースト振動（phase2のみ, OFF1500/ON500/GUARD50ms） | ✅ | `VibrationBurstController.java` |
| IMU生CSV（t_ns, accel xyz, gyro xyz, vib_state, burst_id, phase） | ✅ `{name}_IMU_Data.csv` | `GreenValueAnalyzer.java:2971` |
| meta.json（振動パラメータ＋`sample_rate_hint_hz`） | ✅ | `GreenValueAnalyzer.java:4184-4220` |
| phase状態機械（placement/resting/press/release） | ✅ | `GreenValueAnalyzer.java` |
| 接触面積・press_target・Y/U/V 表示 | ✅ | `MainActivity.java:653-680` |
| Wave_Data に vib_state 等 | ✅ | `GreenValueAnalyzer.java:2943` |

**設計の妙**：振動を**バースト（ON/OFF交互）**にすることで、**OFF窓＝きれいなPPG**、**ON窓＝IMU減衰（VFE）**を同一時計で後から分離できる。実機で「振動してもPPGが汚れない」確認済みの事実とも整合。

## 2. ギャップ（v2との差分）と本日の対応

| ギャップ | 重要度 | 対応 |
| --- | --- | --- |
| 実効IMU Hzが**画面にライブ表示されない**（meta.jsonには有り） | 高 | **本日実装**：UIに `IMU: NNN Hz` 常時表示 → アプリ自体がレート確認ツールになる |
| マイクアクセスON要件の**起動時チェック/警告なし** | 中 | **本日実装**：phase>=1で実効Hz<250なら一度だけ警告表示 |
| `wave_break_score` / `ppg_ac_amp`（波形の崩れ＝SBP目印） | 中 | **アプリ改修不要**：素データ（拍ごと振幅・phase・接触面積・IMU RMS）は記録済み → **PC側解析で算出**（v2 §5方針） |

→ 本日のアプリ改修は**ライブHz表示＋低レート警告＋ビルド確認**の最小3点（Codexへ委譲）。

## 4. 実機検証チェックリスト（Pixel 8・本人作業）

sandboxでは不可。Pixel 8 実機で以下を確認：

1. `cd RealTime-IBI-BP && ./gradlew :app:assembleDebug` がローカルで成功。
2. インストール後、**マイクアクセスをON**にして起動（OFFだとIMUが200Hz制限）。
3. 計測開始 → 画面の **`IMU: NNN Hz` が >200（目標≥400）** を確認（＝VFE成立の前提）。出ない場合は権限/マイクを再確認。
   - **✅ 確認済み（2026-06-27, Pixel 8実機）：`IMU: 531 Hz`** ＝目標≥400をクリア。VFE成立の前提クリア。
   - 同時にSensor Info欄のレイアウトはみ出しを修正（`shrinkColumns="*"`＋最終行を2カラム×2行化）。再ビルド・インストール・実機スクショで確認済み。
4. phase2で**振動が出る**こと、**PPG波形が崩れない**こと（確認済の再現）。
5. セッション終了で `Download/` に **`*_Wave_Data.csv` / `*_Training_Data.csv` / `*_IMU_Data.csv` / `*_meta.json`** が自動保存される。
6. `Analysis/realtime_pipeline/auto_import_sessions.py` で `realtime_sessions/` に取り込まれる（IMU CSV含む）。
7. 押し込みスイープで、後解析時に**PPG振幅が山なり→崩れ**を示すか（オシロメトリ成立の最初の確認）。
