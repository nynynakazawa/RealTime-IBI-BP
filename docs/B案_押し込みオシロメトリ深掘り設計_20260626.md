# B案 深掘り設計 ― 押し込みオシロメトリ（カメラ＋接触面積）の科学的核心（2026-06-26）

B案＝「受動PP＋能動押し込みMAP→絶対BP」の**物理・信号処理・実験設計・難所**を詰める。
最重要の結論を先に：**force-free（接触面積のみ）では x軸が mmHg に校正されないため、“絶対MAP” は最難関**。現実的な主張は2層に分け、**Tier 1（相対MAPトラッキング）を主ターゲット**にする。Tier 2（絶対mmHg・AAMI/BHS）はストレッチ。

---

## 1. オシロメトリの原理（何を物理的に測るのか）

カフ式オシロメトリ：外部圧 P_ext を DBP以下→SBP以上へ上げると、動脈の**拍動性容積振動の振幅**が**ベル型**を描く。
- 振幅が**最大**になるのは **P_ext ≈ MAP**（経壁圧 transmural ≈ 0 ＝動脈コンプライアンス最大）。
- **SBP/DBP** はピーク振幅に対する**固定比（オシロメトリ比, 経験値 例: SBP≈0.5, DBP≈0.7–0.8）**の点として読む。
- 動脈が完全閉塞すると拍動消失（振幅→0）＝ P_ext > SBP。

**指押し版（Mukkamala④）**：カフの代わりに指でカメラ＋画面を押す。経壁圧を押圧で変調し、**振動振幅＝カメラPPG、押圧＝力センサ（ひずみゲージ）÷接触面積**で oscillogram を作り、MAP/SBP/DBP を読む。**x軸は実 mmHg（力を測っているから）**。

**本研究版（force-free）**：力センサを使わず、**接触面積のみ**を圧の代理にする。→ **ここが④との決定的差であり、最大の難所**（下記§2）。

---

## 2. 中心的難問：接触面積 → 絶対 mmHg の変換（force-free の代償）

- ④は P_ext = 力÷面積 を**実測**するので x軸が mmHg。本研究は**力を測らない** → 得られるのは **「振幅 vs 接触面積」**という **x軸が任意単位（接触面積）の relative oscillogram**。
- 接触面積は押圧の**単調だが非線形・未校正・ヒステリシス（粘弾性）**な代理。→ **ピーク位置（=MAP）は“接触面積の値”で出るが、それが何 mmHg かは分からない**。
- したがって**絶対mmHg化には外部参照が要る**。選択肢：
  1. **CNAP教師つき母集団LOSOモデル**：oscillogram形状特徴（ピーク接触面積・幅・左右勾配・非対称・AUC等）→ CNAP MAP を回帰。per-subject校正は無いが**母集団モデル**。
     - **リスク**：接触面積↔実圧の伝達関数が**指サイズ・皮膚・爪・端末レンズ・画面輝度で大きく変わる** → 被験者間分散が巨大 → **受動が絶対で負けたのと同じ壁**に当たる可能性大。
  2. **内部アンカー**：ピーク（MAP）・拍動消失（SBP）・下側膝（DBP）の**形状ランドマーク**は取れるが、**スケール（mmHg）は依然不明**。1点でも絶対圧が分かれば解けるが、force-free では絶対圧の独立アンカーが無い。
  3. **per-user 1回押し校正**：既知参照に1回合わせる → だが**校正フリーが壊れる**ので不採用。
- **帰結（正直に）**：**force-free での絶対mmHg MAP は構造的に難しい**。母集団モデル（選択肢1）に賭けるが、これが between-subject で崩れたら絶対は出ない。

## 3. 主張の2層（ここが深掘りの肝）

絶対（AAMI/BHS）に固執すると受動と同じ壁で全滅する。**主張を2層に分け、現実的な下層を主ターゲットにする。**

### Tier 1（達成可能・相対）＝**主ターゲット**
- **押し oscillogram のピークが MAP の“変化”を追う**（被験者内 ΔMAP トラッキング）。x軸スケール不要・**形状の相対変化**だけ見る。
- 受動PP（ΔPP, 既存）と融合 → **ΔSBP/ΔDBP のトラッキング**が成立。
- **論理（重要）**：受動だけだと ΔMAP が無いので **ΔSBP/ΔDBP は追えない**（既存の確定事実）。押しで **ΔMAP を補えば ΔSBP/ΔDBP が初めて追える**。
- **これは絶対mmHgを一切要求しない**。アブレーション（§6）で「受動のみ ＜ 受動＋押し」を**相対トラッキング指標**で示せば成立。**ECBIOSはこれで十分強い。**

### Tier 2（ストレッチ・絶対）
- oscillogram形状 → **絶対mmHg MAP**（母集団LOSO）→ 受動PP絶対化と合わせ **絶対SBP/DBP**、AAMI/BHS評価。
- between-subject が崩れたら未達 → その場合は**「相対は成立・絶対は今後」**で正直に報告（[投稿計画 GATE2弱](ECBIOS2026_投稿計画_20260626.md)）。

> 方針転換：[投稿計画](ECBIOS2026_投稿計画_20260626.md)はAAMI/BHS（Tier2）を前面に置いていたが、**主ターゲットをTier 1（相対ΔMAP→ΔSBP/ΔDBPトラッキング）に下げる**。絶対は副次。

## 4. 信号処理パイプライン（Claude設計→Codex実装）

押しスイープ1回から oscillogram を作る手順：
1. **拍検出**：押し中のPPG（緑＝Uプレーン平均）で拍ごとに分割。
2. **拍ごとAC振幅**：各拍の peak-to-trough（拍動成分）を算出。**DC変化の補正**：押すと血液が押し出されDC（光路）も変わる → **AC/DC** か **AC を自身の最大で正規化**（自己正規化＝指サイズ・端末非依存）。
3. **x軸＝接触面積**（自己正規化, 単調増加区間のみ採用）。
4. **oscillogram＝振幅 vs 接触面積**を平滑化（spline/LOWESS）。
5. **ランドマーク検出**：
   - **ピーク**（最大振幅）→ MAP点（接触面積座標）。
   - **拍動消失点**（振幅が閾値以下＝閉塞）→ SBP点候補。
   - **下側膝** → DBP点候補。
6. **形状特徴**：ピーク接触面積・ピーク振幅・曲線幅（半値幅）・左右勾配・非対称度・AUC・SBP/DBP点の比。
7. **品質**：押しが非単調/速すぎ/飽和（接触面積が早期に天井）の試行を除外。複数試行で**ピーク再現性**を確認。

## 5. 融合の数学（相対でも成立する論理）

- 恒等式：**SBP = MAP + ⅔·PP, DBP = MAP − ⅓·PP**（MAP≈DBP+⅓PP の近似）。
- 受動 → **PP（相対）**、押し → **MAP（相対）**。
- 融合：ΔSBP = ΔMAP + ⅔·ΔPP, ΔDBP = ΔMAP − ⅓·ΔPP。
- **重要な正直ポイント**：受動PPも**相対（トラッキング）**で絶対mmHgではない。両者が相対なら**融合結果も相対トラッキング**＝Tier 1。絶対SBP/DBP（Tier 2）には**両軸の絶対化**が要る。
- それでも「**受動のみではΔSBP/ΔDBPが追えない → 押しを足すと追える**」は**相対のまま示せる本質的前進**。

## 6. 実験設計：3アーム・アブレーション（前回合意のOption Y）

**1つの新・押しデータ（n=10〜20, CNAP参照）内**で対応比較。唯一の差＝特徴の有無。

| アーム | 特徴 | 期待 |
| --- | --- | --- |
| ① 受動のみ | sys_upslope/cosφ 等 | ΔPPは追うが**ΔSBP/ΔDBPは×**（ΔMAP欠落） |
| ②a 押しのみ | oscillogram形状（MAP） | ΔMAP/ΔSBP/ΔDBPを△〜○ |
| ②b 受動＋押し（融合） | ①＋② | **最良**（ΔMAP+⅔ΔPP） |

- **主張の主軸＝②b vs ①**（受動に押しを足すと ΔSBP/ΔDBP が**有意改善**）。対応のあるLOSO・paired検定・効果量・CI。n小は正直に。
- **②a も入れて3アーム**：逆攻撃「押すだけで足りる＝受動冗長」に対し **②b ≥ ②a**（受動PPが押しの**合間**を埋める付加価値）を示す。
- **相設計**：安静ベースライン → **寒冷昇圧**（MAPを実際に変動させる＝トラッキングの土俵を作る）→ 各相で**押しスイープ複数回**。BPが動かないとトラッキングは検証できないので**寒冷昇圧は必須**。
- **区間定義（交絡注意）**：①受動アームは「押し前の安静/昇圧相の受動特徴」、②は「押しスイープ区間の特徴」。アーム間差が**区間の違い**と交絡しないよう、**同一時間窓・同一参照CNAP値**に揃える設計をCodexに明示。

## 7. 失敗モード・交絡と対策（先に潰す）

| 失敗モード | 中身 | 対策 |
| --- | --- | --- |
| **絶対mmHg化不能** | 接触面積↔実圧が未校正・個人差大 | Tier 1（相対）を主ターゲットに。絶対は母集団LOSOで挑戦し未達なら正直に副次 |
| **粘弾性ヒステリシス** | 指動脈の粘弾性で oscillogram が歪む（④も明記の既知問題） | 押し速度を一定化（UIガイド）、上り/下りを分離、複数試行平均 |
| **接触面積の飽和** | SBP閉塞点に達する前に面積が天井 | スイープ範囲を確保、飽和試行は除外、押し強度ガイド |
| **非単調押し** | ユーザーが等速で押せない | UI速度フィードバック、単調区間のみ採用 |
| **DC変動** | 押下でDC（光路）が変化しAC振幅を汚す | AC/DC正規化、自己最大正規化 |
| **between-subject分散** | 指・皮膚・端末で伝達関数が変わる | 自己正規化（形状のみ）、被験者内トラッキングに軸足 |
| **モーション** | 押し動作自体がアーチファクト源 | コヒーレンスベースのアーチファクト限定除去（生理信号は消さない＝既存の教訓） |
| **n小** | n=10〜20でLOSOは非力 | 効果量・CI・paired検定、絶対主張は控える |

## 8. GATE 1 で見る「最小の信号」（定量基準）

パイロット（n=1〜2）で**B続行可否**を判定する最小条件：
1. 押しスイープで**拍ごとAC振幅が接触面積に対しベル型（単峰）**を描き、**ピークが検出可能**。
2. そのピーク（=MAP点）が**複数試行で再現**（接触面積座標の分散が小さい）。
3. **寒冷昇圧でCNAP MAPが上がるとき、oscillogramピークの形状/位置が対応して動く**（ΔMAPトラッキングの芽）。
- **3が立てば Tier 1 はほぼ射程**。1〜2のみで3が出ないなら、相設計や押し方を見直し（それでもダメならB撤退→A案）。

## 9. ④（Mukkamala）との厳密な差分（新規性の線引き）

| 項目 | ④ 指押しオシロメトリ | 本研究B |
| --- | --- | --- |
| 圧軸 | **力センサ（ひずみゲージ）÷面積＝実mmHg** | **接触面積のみ（force-free, 任意単位）** |
| 出力 | 絶対SBP/DBP | **Tier1=相対ΔSBP/ΔDBPトラッキング**（絶対は副次） |
| 受動との関係 | 押し単独 | **受動PPと融合（PP/MAP分解の役割分担）** |
| 評価 | 単発測定 | **寒冷昇圧下トラッキング・3アームアブレーション** |
| 参照 | カフ/指カフ | **CNAP連続** |

→ **新規性の核は「force-free接触面積オシロメトリ」＋「受動PPとのモジュール融合」＋「摂動下トラッキングのアブレーション」**。**“押すこと”自体は④で既出**なので主張しない（[新規性層別](接触カメラPP先行研究比較_20260625.md)）。

## 10. 残る未知数と最短 de-risk 順序

1. **（最優先）oscillogram がそもそも出るか**：M0実機検証後すぐパイロット。ベル型・ピーク検出（§8の1,2）。
2. **ΔMAPトラッキングが立つか**：寒冷昇圧でピークが動くか（§8の3）＝GATE 1本体。
3. **アブレーションで②b>①が有意か**：本収集後（GATE 2）。Tier 1の本丸。
4. **（賭け）絶対mmHg化**：母集団LOSOが between-subject で耐えるか（Tier 2）。

**この順で、立たなくなった時点で主張を1段下げる**（絶対→相対→A案フォールバック）。各段で投稿は守られる。

## 11. 一行サマリ
**force-freeの代償でx軸がmmHg化されないため絶対MAPは最難関。主張を2層に分け、絶対(Tier2)ではなく相対ΔMAP→ΔSBP/ΔDBPトラッキング(Tier1)を主ターゲットにする。「受動だけではΔSBP/ΔDBPを追えない／押しでΔMAPを補うと追える」を、寒冷昇圧下・3アームアブレーション・対応LOSOで示すのがBの核心。新規性は〈force-free接触面積オシロメトリ＋受動PPとの融合＋摂動下トラッキング〉に限定し、“押すこと”自体（④既出）は主張しない。**
