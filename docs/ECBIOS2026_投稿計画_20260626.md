## 3. 押しデータ収集プロトコル（設計：Claude→Codex/実機）

### 3.1 計測条件

- 端末：実装済みアプリ（背面カメラ＋緑照射、Y/U/V平均＋接触面積/重心/長短径/phase、約30fps）。
- 参照：**CNAP連続血圧**（SBP/DBP/MAP）。押し区間とPPG/接触面積を**共通タイムスタンプで同期**。
- 被験者：パイロット n=1〜2 → 本収集 n≥8〜10（既存12名と整合する範囲で可能なら再利用）。
- 相：①安静ベースライン → ②**押し込みスイープ**（ガイド一定速度で接触面積↑）→ ③解放 を1試行。寒冷昇圧相も別途。
- 試行数：1セッション内で押し込みを複数回（包絡再現性の確認）。

### 3.2 押し込みマニューバ（再現性が命）

- **ガイド一定速度**で接触面積を単調増加（UIで速度ガイド）。力較正はしない（可搬性優先）。
- 接触面積は**計測内で自己正規化**（指サイズ非依存の包絡形を見る）。
- 光学アンカー：**振幅ピーク＝MAP、脈波消失＝SBP**（オシロメトリ原理）を仮説として検証。

### 3.3 取得・品質

- meta.json（YUV→RGB式・閾値・phase定義）を毎回保存。
- 品質ゲート：Y_mean[35,245]、接触%、touch_valid。
- 自動取込：`Analysis/realtime_pipeline/auto_import_sessions.py` で merge。

## 4. 解析プラン（Claude設計→Codex実装）

### 4.1 既存データの宿題（A案強化・M0b・即着手）

- **PP絶対MAE/MAPE**と**ΔPP MAE**を、**被験者平均予測ベースライン併記**でLOSO算出（[接触カメラPP比較 §4.5](接触カメラPP先行研究比較_20260625.md) の指摘に対応）。
- 既存図5枚（fig1〜5）を論文品質に再出力（誤差棒・キャプション）。

### 4.2 押しデータ（B案本体）

1. **包絡抽出**：接触面積（圧代理, 自己正規化）に対する拍ごとPPG振幅の包絡を作る。
2. **GATE 1判定**：包絡ピークの接触面積位置 ↔ CNAP MAP の対応（被験者内）。相関・再現性。
3. **絶対化**：包絡ピーク→MAP推定、脈波消失→SBP推定（オシロメトリ係数をCNAP参照でLOSO学習）。
4. **融合**：能動MAP ＋ 受動PP（sys_upslope/cosφ）→ **MAP+⅔PP=SBP, MAP−⅓PP=DBP**で絶対BP再構成。
5. **付加価値検証（最重要）**：「押し単独BP」vs「押しMAP＋受動PP」で、**受動PPが連続区間で押し単独を上回るか**（[同 §4.6.3 条件3](接触カメラPP先行研究比較_20260625.md)）。
6. **基準準拠評価**：AAMI/BHS、Bland-Altman、LOSO。

## 5. アブストラクト設計（8/15提出物）

方針：**A案の安全な物語**を土台に、**パイロットが立てば能動MAPを「本研究の方法」として予告**。立たなくても成立する文面にする。

骨子（150〜250語想定, 確定後に英訳）：

1. **背景**：校正フリー・接触・30fps可視スマホでの絶対BPは未達。なぜかは未解明。
2. **方法/発見1（既存・確実）**：被験者内でBPをPP/MAPに分解。**PP（脈圧）は解釈可能な収縮期立ち上がり特徴（sys_upslope≡cosφ）で追従可（cold pressor, LOSO, p<0.01）だが、MAP/絶対は校正フリーでは構造的に不能**。
3. **方法2（B案予告）**：欠けたMAP軸を**能動押し込みオシロメトリ（接触面積を圧代理, 力センサ不要）**で取得し、受動PPと融合して絶対BPを再構成するハイブリッドを提案。
4. **結果**：（パイロットが立てば）押し包絡ピークがCNAP MAPに対応する予備結果。（立たなければ）PP/MAP分解と方法論的教訓（外れ値除去が生理信号を消す）を報告。
5. **結論**：最小構成で何が回復可能かの境界を引き、能動マニューバで絶対BPへ向かう道筋を示す。

※精度（MAE/MAPE）は主張の前面に出さなBい（[比較 §4.5](接触カメラPP先行研究比較_20260625.md)）。
